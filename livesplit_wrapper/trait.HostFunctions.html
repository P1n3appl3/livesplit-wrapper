Expand description

The autosplitter’s interface for interacting with the LiveSpilit timer.

Provided methods

[src]

fn print(&self, str: &str)

Output a message. This can be used for debugging and/or sending error messages to the player through whichever LiveSplit frontend they’re using. Note that because autosplitters run in WASM, they don’t have access to STDOUT or files, so typical solutions like println! and logging will not work (this could chagne in the future as LiveSplit plans to support WASI).

fn attach(&self, name: &str) -> Option<Process>

Attach to a process running on the same machine as the autosplitter.

[src]

fn start(&self)

Start the timer for a run. Note that this will silently do nothing on subsequent calls. To start a new run, call reset() and then start().

[src]

fn pause(&self)

Pause the game time counter. This is often used when entering a loading screen or end level screen for games that use in game time rather than real time. It may be a good idea to call set_game_time() immediately after pausing so that LiveSplit’s game time counter shows the exact current time.

fn reset(&self)

Reset the run. Don’t do this automatically when a run has finished, and in general be conservative about resetting runs from the autosplitter. Common practice is to do so only if there’s an unambiguous signal that the player is done with this run.

fn set_game_time(&self, time: f64)

Set the game time. Note that if the timer is not paused, the time shown will keep incrementing immediately after it is set to the given value.
